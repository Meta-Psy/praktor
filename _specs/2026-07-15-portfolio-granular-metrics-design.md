# Praktor Portfolio — гранулярные метрики, полу-авто счётчики, свежесть

**Дата:** 2026-07-15
**Статус:** DRAFT — ждёт approve Alex
**Проект:** praktor (`mc.alexmetapsy.com`, S1 Mission Control)
**Предшественник:** `_specs/2026-06-11-phase-f-stage-s1-portfolio-dashboard-design.md` (v1, бинарные направления)

## Зачем

Praktor становится **основой контроля всех проектов** Alex. Текущий дашборд `/portfolio`
показывает направления бинарно (`planned|doing|done`) и считает `percent` как долю
`done`. Этого мало по трём причинам:

1. **Нельзя выразить «33 из 2250»** — прогресс контент-фабрики (2102 конспекта, 33750
   тестов, 34 видео…) не бинарен.
2. **Данные протухают незаметно** — на проде `portfolio.json` от 2026-06-14, единственный
   публиш был «initial»; roadmap-блоки описывают майское состояние (PR #64/#58). Смотрящий
   не понимает, что видит месячную давность.
3. **Нет иерархии** — контент-фабрика это ~11 подпродуктов, каждый со своим счётчиком; на
   уровне «проект» их не разложить.

## Решения Alex (2026-07-15, зафиксированы)

| Развилка | Выбор |
|---|---|
| Модель прогресса | **Иерархия** проект → подпроект → метрика |
| Источник цифр | **Полу-авто** из реального сигнала (glob/команда) при публикации; ручной — с датой |
| Свежесть | **Дата актуальности + индикатор протухания** (жёлтый бейдж старше порога) |

## Что НЕ делаем (граница)

- Нет истории/burndown/графиков динамики (свежесть = один факт «на дату», не ряд).
- Нет запроса к прод-БД из публишера напрямую (креды в публишер не тащим — см. `command`-источник).
- Не ломаем v1: блоки только с `directions` рендерятся как раньше.
- Не трогаем live-чип CI/деплой (`/api/projects`, `mc_key`) — он ортогонален.

## Архитектура

Три слоя, как в v1: **roadmap-блок в памяти** → **публишер (`portfolio-publish.cjs`)** →
**`portfolio.json` в `Meta-Psy/portfolio-data`** → **Go API (`portfolio.go`)** →
**фронт (`Portfolio.tsx` / `portfolioStatus.ts`)**.

### Схема roadmap-блока v2 (в `project_*.md`)

Аддитивно к v1. Новое поле `subprojects`; `directions` остаётся для бинарных этапов.

```jsonc
{
  "name": "Контент-фабрика Mentis",
  "status": "active",
  "next_action": "Био-9 §19-46 конспектами (корпус готов)",
  "mc_key": "mentis",
  "directions": [                          // v1, необязательно — бинарные этапы
    { "title": "ТЗ approved", "state": "done" }
  ],
  "subprojects": [                         // v2 — иерархия
    {
      "key": "conspects",
      "label": "Конспекты",
      "metrics": [
        {
          "key": "all",
          "label": "все языки",
          "unit": "док",
          "total": 2102,
          "weight": 1,                     // необязательно, для роллапа; default 1
          "source": {                      // ← НЕ попадает в portfolio.json (см. ниже)
            "type": "glob",
            "root": "~/OneDrive/Desktop/Mentis",
            "pattern": "Конспекты */**/*.html",
            "exclude": "_Архив*/**"
          }
        }
      ]
    },
    {
      "key": "video",
      "label": "Видео",
      "metrics": [
        { "key": "basic", "label": "Basic RU", "unit": "ролик",
          "done": 0, "total": 34, "source": { "type": "manual" }, "as_of": "2026-07-13" }
      ]
    }
  ]
}
```

### Типы источников (`source`) — резолвятся публишером

| type | что делает | `done` откуда | `as_of` |
|---|---|---|---|
| `glob` | считает файлы под `root` по `pattern` (минус `exclude`) | число совпадений | штампуется `generated_at` |
| `command` | запускает `run` в `cwd`, парсит первое целое из stdout | это целое | штампуется `generated_at` |
| `manual` | ничего | берётся литерал `done` из блока | берётся литерал `as_of` (обязателен) |

`total` — обычно литерал (цель). Может иметь свой `source` тем же механизмом (например,
`total` = число тем в КТП через `command`). `weight` — для роллапа, по умолчанию 1.

**`command` — граница доверия:** это произвольное исполнение shell при публикации. Допустимо
ТОЛЬКО потому, что roadmap-блоки — авторские файлы памяти Alex на его машине. В публичный
`portfolio.json` спека источника не попадает.

### Резолв в публишере (`portfolio-publish.cjs`)

1. Читает блок, валидирует по расширенному белому списку.
2. Для каждой метрики резолвит `source` → получает `done` (+ `as_of` для авто).
3. **Вырезает `source` из выходного объекта** — в `portfolio.json` уезжают только
   `{ key, label, unit, done, total, as_of }`. Причина: `portfolio-data` — внешний репо,
   локальные пути (`/c/Users/Alex/...`) утекать не должны (усиливает существующий PII-фильтр).
4. Ошибка резолва одной метрики → WARN в stderr + метрика помечается `"error": true`
   (не роняет весь публиш), `as_of` не обновляется.

### Роллап и проценты (`portfolioStatus.ts`)

Единицы у метрик разные (док vs вопросов vs роликов) — **суммировать `total` между
единицами нельзя** (соврёт). Поэтому:

- **Метрика %** = `round(done/total*100)`.
- **Подпроект %** = взвешенное среднее % его метрик (вес = `weight`).
- **Проект %** = взвешенное среднее % подпроектов; если подпроектов нет — старый бинарный
  `percent(directions)`.

Так честно при смешанных единицах и обратно совместимо.

### Свежесть (`portfolioStatus.ts` + `Portfolio.tsx`)

- Каждая метрика несёт `as_of` (ISO-дата).
- Фронт: `staleDays(as_of)` = дней от `as_of` до сейчас. Если `> STALE_THRESHOLD_DAYS`
  (константа, старт **21**) → жёлтый бейдж «⚠ Nд» рядом с числом.
- Это независимо от существующего `doc.stale` (тот про сбой дозагрузки из репо).

### Backend (`internal/web/portfolio.go`)

Аддитивно:

```go
type Metric struct {
    Key   string  `json:"key"`
    Label string  `json:"label"`
    Unit  string  `json:"unit,omitempty"`
    Done  int     `json:"done"`
    Total int     `json:"total"`
    AsOf  string  `json:"as_of,omitempty"`
    Error bool    `json:"error,omitempty"`
}
type Subproject struct {
    Key     string   `json:"key"`
    Label   string   `json:"label"`
    Metrics []Metric `json:"metrics"`
}
// PortfolioProject += Subprojects []Subproject `json:"subprojects,omitempty"`
```

API — сквозной проброс (парсит JSON, отдаёт). Тест: гарантия, что `source` в ответе не
появляется (его нет в типах — структурно невозможно, но проверим на реальном фикстуре).

### Frontend (`Portfolio.tsx`)

В раскрытой карточке проекта под лейнами направлений — секция подпроектов: на каждый
подпроект заголовок + список метрик, у метрики мини-бар (тот же рисованный `div`, что уже
есть), подпись `done/total unit`, `% `, дата и stale-бейдж. Chart-либы не добавляем
(прецедент — самодельный SVG/`div`-бар).

## Риски

- **Смешанные единицы в роллапе** — снят выбором «среднее %», не сумма totals.
- **`command` = произвольный shell** — принят как граница доверия (авторские файлы Alex).
- **Пути в публичном репо** — снят вырезанием `source` из выхода.
- **glob по OneDrive/Desktop** — путь с пробелами и кириллицей; резолвер обязан работать с
  ними (тест на кириллический путь).
- **Роллап-честность vs ожидания** — «проект 40%» это среднее %, а не «40% работы»;
  задокументировать в подписи/тултипе, чтобы не вводило в заблуждение.
